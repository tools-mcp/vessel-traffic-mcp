# Code of Conduct

`vessel-traffic-mcp` welcomes contributors who help build a read-only,
source-attributed maritime MCP server without compromising provider
terms, operator credentials, or user safety.

## Expected Behavior

- Be respectful, direct, and constructive in issues, discussions, and
  pull requests.
- Keep technical disagreement focused on evidence, tradeoffs, and
  maintainability.
- Respect provider terms, rate limits, access controls, and data-source
  attribution.
- Report security issues privately through `SECURITY.md`.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory conduct.
- Publishing another person's private information or credentials.
- Posting API keys, bearer tokens, cookies, raw HAR files, private
  captures, or replayable session data.
- Asking contributors to bypass authentication, paywalls, CAPTCHA, bot
  defenses, rate limits, or access controls.
- Presenting AIS data from this project as safety-critical navigation
  data.

## Enforcement

Maintainers may edit or delete comments, close issues or pull requests,
or block participants who violate this code. Security-sensitive reports
should be moved to the private process described in `SECURITY.md`.
